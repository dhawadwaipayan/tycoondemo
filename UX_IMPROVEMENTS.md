# Tycoon US Rebuild — What Improved

This document lists the concrete improvements in the UI rebuild versus the earlier Tycoon US product (Astra · Work · Library · Team).

---

## 1. Product framing

| Before | After | Improvement |
|--------|--------|-------------|
| Felt like a **chatbot** with side pages | Feels like an **AI company workspace** | Product matches the “AI team” promise |
| Astra *was* the app | Astra **coordinates** inside the app | Clearer role; system is bigger than one chat |
| Success = “Astra replied” | Success = “work moved, decisions surfaced, outputs exist” | Outcome-oriented UX |

**Bottom line:** The rebuild improves *what Tycoon is*, not only how it looks.

---

## 2. Information architecture

### Before
```
Astra → Work → Library → Team → Settings
```

### After
```
Home → Chats → Tasks → Schedule → Team → Library
```

| Improvement | Why it matters |
|-------------|----------------|
| **Home** as default land | Founders start with command + pulse, not an empty/long thread |
| **Chats** instead of “Astra-only” nav | Supports 1:1 agents + group initiatives, not one mega-thread |
| **Tasks** instead of “Work” list | Execution is a first-class system of record |
| **Schedule** in the IA | Recurring work has a home (not only re-asking in chat) |
| **Recent** in the sidebar | Fast return to active projects |
| Org header + Share | Workspace / company feel, not consumer chat |

---

## 3. Mental model

| Before | After |
|--------|--------|
| Talk to Astra | Run your AI company |
| Work = Astra’s queue | Tasks = all work (you, Astra, agents) |
| Library = handbook + files | Library = memory + artifacts of work |
| Team = roster | Team = people, agents, control |
| Chat holds everything | Chat communicates; Tasks track; Home summarizes |

**Core loop (new):**

```
Ask (Home/Chat) → Assign (Astra/agents) → Track (Tasks) → Store (Library) → Review (Home / Waiting on you)
```

**Improvement:** Users learn six clear layers instead of “everything lives in the Astra thread.”

---

## 4. Home (new surface)

**Before:** No real command center — Astra chat was the landing experience.

**After — Home includes:**

- Command composer (“What can I do for you?”)
- Quick create chips (Table, Doc, Slides, Board, Calendar, Timeline, Checklist)
- “What’s happening now” agent cards (status + CTA)
- Compact task board snapshot
- Templates (Product Launch, Brand Identity, Competitor Analysis…)

| Improvement | Detail |
|-------------|--------|
| Faster time-to-action | Type or tap; no hunting for Astra first |
| Company pulse | See Indie / Riley / Darren updates without opening each chat |
| Work visibility on day one | Board snapshot on Home |
| Guided starts | Templates reduce blank-page anxiety |
| Brand presence | Editorial hero + atmosphere (not flat empty chat) |

---

## 5. Chats (rebuilt from Astra page)

| Before (Astra) | After (Chats) | Improvement |
|----------------|---------------|-------------|
| Single primary thread | Pinned + Groups + Agents directory | Parallel work is navigable |
| Integrations strip as chrome | Focus on conversation + work objects | Less distraction |
| Credit plan cards in the stream | Task spawn cards + tool-use lines | Trust work, don’t interrupt with checkout |
| Text-only replies | Structured embeds (“3 Tasks Created”, task ID/status) | Chat creates operable objects |
| No side document | Resizable doc editor beside chat | Discuss and draft in one place |
| Opaque AI actions | `Used tyctl task create` style disclosure | Autonomy is visible |
| One conversation metaphor | Browser-like tabs for parallel threads | Matches real founder multitasking |

**Improvement summary:** Chat becomes a **communication + spawn layer**, not the whole product and not a billing surface.

---

## 6. Tasks (rebuilt from Work page)

| Before (Work) | After (Tasks) | Improvement |
|---------------|---------------|-------------|
| “Astra’s current work” | Workspace-wide tasks | Includes user- and agent-spawned work |
| Ongoing / Delivered only | Backlog · In Progress · Live · Completed | Real stage granularity |
| Thin rows (ID, title, schedule, Paused) | Cards with owner, priority, estimate | Scannable ownership |
| “Needs you” banner only | **Waiting on you** filter with badge | Approvals are a durable mode |
| No source filters | Spawned by me / Spawned by Astra | Clear provenance |
| No project slicing | Tabs (All / AcmeAI Launch / Marketing Prep) | Focus by initiative |
| No detail view | Right-side task drawer | Inspect without leaving the board |
| List-only | Board + List toggle | Operator choice |

**Improvement summary:** Work is no longer a status list — it is an **execution board** founders can run.

---

## 7. Layout & workspace craft

| Before | After | Improvement |
|--------|--------|-------------|
| Single-column pages | Left rail + context panel + main + optional inspector | Desktop OS feel |
| Static full sidebar always | Collapsible rail (auto-collapses on Chats) | More space for chat + doc |
| Flat white pages | Warm rail + inset paper workspace | Depth and hierarchy |
| Page = destination | Panels resize; drawers open in context | Stay in flow |

**Improvement summary:** From a **website of pages** to a **multi-pane workspace**.

---

## 8. Trust, control & autonomy

| Improvement | How it shows up |
|-------------|-----------------|
| Visible ownership | Assignees on every task card |
| Visible coordination | Astra spawn cards listing Riley / Jordan / Casey |
| Visible tooling | Tool-use lines in chat |
| Visible attention | Badges, “decisions need review”, Waiting on you |
| Guardrails in copy | “I’ll ask before publishing or contacting anyone” |
| Founder control | Review / Resolve CTAs on agent cards |

**Improvement summary:** AI work feels like **employees you can manage**, not a black box.

---

## 9. Attention & hierarchy

| Priority | Before | After |
|----------|--------|--------|
| Needs you | Orange banner on Work | Home cards + Chats pin + Tasks filter |
| In flight | Mixed into Ongoing list | In Progress / Live columns + High pills |
| Done | Delivered tab | Completed column (muted) |
| Create | Bottom composer on Astra | Home composer + quick actions + templates |
| Billing | Hero cards inside chat | Intentionally de-emphasized in the work stream |

**Improvement summary:** The UI tells you **what matters first** instead of competing modules fighting for the first viewport.

---

## 10. Design & brand uplift

| Area | Improvement |
|------|-------------|
| Typography | Cirka (display) + Gilroy (UI) — editorial, brandable |
| Color | Warm paper shell, ink CTAs, mint/sepia atmosphere washes |
| Motion | Typewriter hero, staggered cards, route transitions — presence, not noise |
| Icons | Consistent Phosphor weights |
| Aesthetic | Avoids generic purple-glow “AI SaaS” look |
| Home brand test | First viewport still feels like Tycoon without the nav |

**Improvement summary:** The rebuild is usable as both a **product UX proposal** and a **website visual direction**.

---

## 11. Cross-surface continuity

| Before | After |
|--------|--------|
| Chat, Work, Library felt disconnected | Same agents, tasks, and CTAs repeat across Home → Chats → Tasks |
| Hard to follow “who did what” | Spawn → board → card → drawer is one path |
| Outputs easy to lose in threads | Doc panel now; Library planned as durable store |

**Improvement summary:** One **object model** (chat ↔ task ↔ agent ↔ artifact) instead of three separate apps.

---

## 12. What this rebuild improves for the business / website

1. **Proof matches the claim** — “AI company” is visible in IA and UI, not only marketing.  
2. **Demoable story** — Home → Chats → Tasks in under five minutes.  
3. **Clearer differentiation** — Not “another ChatGPT wrapper”; an operating workspace.  
4. **Healthier monetization UX** — Credits/billing don’t hijack the primary work narrative.  
5. **Handoff-ready** — Specs (`prd.md`, `design.md`, `UX_PROPOSAL.md`) + live prototype.

## 14. Short summary

**The rebuild improves Tycoon by making work, not chat, the center of gravity** — with a founder Home, operable Tasks, structured Chats, visible agents, clearer hierarchy, and a premium brand system that can sell the product on the website as well as in the app.
